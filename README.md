# Sibra SOC Dashboard

## Disclaimer
This application is a simulation tool designed for training and demonstration purposes only. All network telemetry, incident data, and threat alerts generated by this dashboard are pseudo-randomly produced by an internal simulator engine and do not reflect real-time traffic from any actual network infrastructure.

## Live Demo
You can view the live demonstration of this dashboard at: [https://soc-analyst-simulation.vercel.app/](https://soc-analyst-simulation.vercel.app/)

## Overview
The SOC Dashboard is a simulated security monitoring interface tailored for Security Operations Center environments. It presents real-time visualization of network activity, threat detection metrics, and incident response workflows. The dashboard is designed to emulate how analysts monitor alerts, investigate anomalies, and manage security incidents efficiently in a controlled simulation environment.

## Features
- Real-Time Telemetry Feed: Monitors incoming network logs with detailed metadata including source IP, target system, and confidence scores.
- Advanced Threat Simulation: Simulates various attack types such as DDoS, SQL Injection, Brute Force, and Ransomware.
- Multi-Modal Reporting: Supports both automated system reports and analyst-verified manual reports.
- PDF Export: Allows analysts to export incident reports directly to PDF format for official documentation.
- Dual-Theme Interface: Fully responsive design with support for both Dark and Light modes.
- Performance Focused: Built with vanilla JavaScript and optimized CSS for maximum responsiveness.

## Technology Stack
- Backend: Python 3.14 with Flask
- Frontend: HTML5, Vanilla CSS3, JavaScript
- Styling: Bootstrap 5, Font Awesome 6
- Libraries: Marked.js (Markdown parsing), HTML2PDF.js (PDF generation)

## Installation and Setup

### Prerequisites
- Python 3.10 or higher
- pip (Python package manager)

### Local Setup
1. Clone the repository to your local machine.
2. Navigate to the project directory.
3. Install the required dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Run the application:
   ```bash
   python app.py
   ```
5. Open your browser and navigate to `http://localhost:5000`.

## Deployment
This project is configured for deployment on Vercel as a Serverless Function. It includes a `vercel.json` configuration file (if applicable) and utilizes a modular architecture compatible with WSGI standards.

## Security Considerations
- Use secure environment variables in production environments.
- Regularly update dependencies listed in `requirements.txt`.
